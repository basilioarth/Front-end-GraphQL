## Funcionalidades entregues e 100% funcionais

### Autenticação completa
- Login + Cadastro (com validação no back-end)
- Token JWT + Refresh Token
- Zustand com `persist` (mantém usuário logado)
- Logout completo (limpa store + Apollo cache + redireciona)

### Apollo Client configurado perfeitamente
- `authLink` → Bearer Token automático em todas as requisições
- `errorLink` preparado (pronto para usar)
- `ApolloProvider` envolvendo a aplicação
- Cache atualizado automaticamente

### Página de Ideias (/)
- Criação de ideia (modal + mutation)
- Listagem completa com `useQuery`
- Detalhe da ideia em Drawer (useLazyQuery)
- Exibição de comentários e votos
- Botão de votar e comentar (estrutura pronta – só falta conectar as mutations finais)

### Página de Membros (/members)
- Listagem completa de usuários (`listUsers`)
- Busca visual (input pronto)
- Convite de novo membro (mutation `createUser`)
- Edição de membro (mutation `updateUser`)
  - Alteração de nome e role (Admin / Member / etc.)
- Remoção de membro (mutation `deleteUser` com confirmação)
- Refetch automático após create/update/delete

### Componentes finais criados
- `EditMemberDialog.tsx` → edição de nome e permissão
- `DeleteMemberDialog.tsx` → confirmação + destructive button
- `MemberCard.tsx` → avatar, role badge, “Você”, botões de ação

### UI/UX final
- Header completo com:
  - Logo + navegação (Ideias / Membros)
  - Avatar + nome + e-mail + botão logout
- Todas as telhas exatamente como o protótipo
- Feedback com Sonner Toast em todas as ações
- Loading states em botões e modais

## Tecnologias usadas (stack completa)

## Problemas clássicos do REST resolvidos com GraphQL
- Zero overfetching/underfetching
- Uma única query por tela (ex: listIdeas já traz autor, votos, comentários)
- Atualizações em tempo real com refetch/cache update
- Tipagem forte com TypeScript + GraphQL
